# Cross-Model-Transferability-of-Hallucination-Signals
This repository contains code for studying whether output-side hallucination signals transfer across different large language model families. The project uses CommonsenseQA as a closed-answer evaluation setting.

For each model response, the code extracts four output-side features which include entropy, top-2 probability margin, length-normalized log probability, and selected answer token count. These features are used to train a lightweight MLP and evaluate whether hallucination detectors trained on one model can generalize to outputs from another model.

The experiments compare single-feature baselines against a multi-signal MLP detector across Llama, Mistral, and Gemma model outputs.

A more indepth analysis can be found in the <a href="./Cross-Model_Transferability_of_Hallucination_Signals_in_LLMs.pdf" target="_blank">Cross-Model Transferability of Hallucination Signals in Large Language Models</a> paper.

Original models and the dataset can be found below:
- CommonsenseQA: https://huggingface.co/datasets/tau/commonsense_qa
- Gemma 2: https://huggingface.co/google/gemma-2-9b-it
- Mistral: https://huggingface.co/mistralai/Mistral-7B-Instruct-v0.3
- Llama 3.1: https://huggingface.co/meta-llama/Llama-3.1-8B-Instruct

## Installation Setup
  - Install Python `3.12.12` and the necessary packages from `requirements.txt`
    - conda can be used to easily manage the environment:
      - `conda create --name <name> python=3.12.12`
      - `conda activate <name>`
      - `pip install -r requirements.txt`
   
  - Make sure to create a HuggingFace Access Token
    - This is required to gain access to certain models
    - When access is granted models and datasets will be downloaded automatically when running `llm_inference.py`

  - Developed under Windows with a GeForce RTX 5090, macOS and Linux compatibility were not tested.

## Reproducibility and Usage
The main workflow is split across four files:

1. `llm_inference.py`
2. `baseline_results.py`
3. `mlp_classifier.py`
4. `hallucination_demo.ipynb`

The files should be ran from the root directory of the repository.

### 1. Run LLM inference and feature extraction

```bash
python llm_inference.py
```

This file loads CommonsenseQA, formats each question as a multiple-choice prompt, scores each answer option using the selected LLMs, and extracts output-side features from the model scores.

The extracted features are:

- entropy
- top-2 probability margin
- length-normalized log probability
- selected answer token count

Additionally, the script runs inference for Llama, Mistral, and Gemma on the CommonsenseQA train and validation splits.

Example output files that are generated:

```text
outputs/features_Meta_Llama_3_8B_Instruct_train.csv
outputs/features_Meta_Llama_3_8B_Instruct_validation.csv
```

Each CSV contains the original question, answer choices, gold answer, model prediction, correctness label, and extracted output-side features.

The repository already includes the generated output files from the original inference runs for reproducibility. These files allow the baseline and MLP experiments to be rerun without repeating the full LLM inference step.

Additional model families can be tested by adding new model entries to `MODEL_CONFIGS` and rerunning `llm_inference.py`. The resulting feature CSVs can then be used with the same baseline and MLP evaluation pipeline.

### 2. Run single-feature baseline experiments

```bash
python baseline_results.py
```

This file trains logistic regression baselines using one feature at a time. The baseline feature sets are:

- entropy only
- top-2 margin only
- length-normalized log probability only
- selected answer token count only

Each baseline is trained on one model's training outputs and evaluated on each model's validation outputs. The output contains accuracy, F1, precision, recall, and AUROC for each train-model and test-model pair.

### 3. Train and evaluate the multi-signal MLP

```bash
python mlp_classifier.py
```

This file trains the MLP detector using all four output-side features together. It evaluates both same-model and more importantly cross-model transfer settings.

Expected output file:

```text
outputs/mlp_results.csv
```

The output contains accuracy, F1, precision, recall, and AUROC for each train-model/test-model pair and the combined experiments.

Model checkpoints for the trained MLP detectors are saved in:

```text
demo_files/
```

Each checkpoint contains the trained model weights, feature order, scaler statistics, and validation AUROC information.

### 4. Run the demo notebook

Open:

```text
hallucination_demo.ipynb
```

The notebook provides a small demo using a couple examples and a trained MLP checkpoint. It is setup using extracted feature examples from Gemma 2. Before running the notebook, make sure the required checkpoint files has already been generated by running:

```bash
python mlp_classifier.py
```

The notebook should produce detector predictions for selected examples and show how the trained MLP uses the extracted output-side features to correctly predict hallucinated responses.
